  3. 输入react-native run-android报红屏错误


    解决方法：发现我的安卓虚拟机没联网，之前也报过这个错误，回去看了发现上面1，2两个错。1和2都解决完再连上虚拟机网络就好了。

  4. 错误3之后，重新输入react-native run-android，报错解决方法：发现之前react-native start并没有完成，而是卡在了Loading dependency graph, down.

    网上有人说是因为端口占用问题，我按照这个方法发现8081端口正是node.exe在运行，所以并不是这个问题。尝试进入这个网站后，cmd出现
    卡在了98%，
    大概是说不存在一个文件。由于我这是github上下载的rn项目，去这个项目的评论区看到别人也有这个错，原来是因为该项目是在mac上编写的，转到windows上就出错了。手动创建报错里的文件就解决了。会连续报错三次，创建了三个文件。
